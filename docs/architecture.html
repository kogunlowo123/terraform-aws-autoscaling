terraform-aws-autoscaling Architecture

HCL / Terraform AWS Infrastructure / IaC

Executive Summary

This module provisions AWS Auto Scaling Groups with launch templates and comprehensive scaling capabilities including target tracking, step scaling, predictive (ML-driven) scaling, warm pools, mixed instances policies (on-demand/spot), lifecycle hooks, instance refresh for rolling deployments, scheduled actions, and SNS notifications. It enables organizations to automatically right-size compute capacity, reduce costs through spot integration, and achieve zero-downtime deployments with rolling updates and auto-rollback.

Data Flow

  1. The Launch Template defines the instance blueprint: AMI, instance type, IAM role, user data, security groups, and monitoring settings.
  2. The Auto Scaling Group uses the template to launch EC2 instances across configured subnets (multi-AZ).
  3. When mixed instances policy is configured, the ASG allocates a base of on-demand instances and fills remaining capacity with spot instances using the specified allocation strategy.
  4. Scaling policies monitor CloudWatch metrics and adjust the desired capacity: target tracking maintains a metric target (e.g., 50% CPU), step scaling applies graduated adjustments, and predictive scaling uses ML to forecast demand.
  5. Warm pool maintains pre-initialized instances in Stopped or Hibernated state for rapid scale-out.
  6. Instances register with ALB target groups, and the health check type (EC2 or ELB) determines instance health.
  7. Lifecycle hooks pause instance transitions, allowing custom scripts (via Lambda/SSM/SNS) to run before launch or termination completes.
  8. Instance refresh performs rolling deployments, replacing instances in batches while maintaining a minimum healthy percentage, with optional auto-rollback.
  9. SNS notifications alert on launch, termination, and error events for operational visibility.

Security Controls

  • IAM Instance Profiles — Configurable IAM role attachment for least-privilege EC2 access.
  • Security Groups — Network-level access control via security group associations.
  • Detailed Monitoring — Enabled by default for 1-minute CloudWatch metrics.
  • Tag Propagation — Tags propagated to instances and volumes for cost allocation and compliance.
  • Create-Before-Destroy — Lifecycle policy ensures zero-downtime during template updates.
  • Instance Maintenance Policy — Controls min/max healthy percentages during maintenance operations.
  • Auto-Rollback — Instance refresh can automatically rollback failed deployments.
  • Scale-In Protection — Configurable protection for instances during instance refresh.

Production Readiness Checklist

  • Set health_check_type = "ELB" when using ALB target groups
  • Configure an appropriate health_check_grace_period for application startup time
  • Enable detailed monitoring (enable_monitoring = true, default)
  • Attach an IAM instance profile with least-privilege permissions
  • Use mixed instances policy for cost optimization with spot instances
  • Configure warm pool for latency-sensitive scale-out scenarios
  • Set up instance refresh with auto-rollback for safe deployments
  • Configure lifecycle hooks for graceful drain/registration
  • Set up SNS notifications for operational alerting
  • Deploy across multiple subnets in different AZs for high availability
  • Enable predictive scaling for workloads with recurring traffic patterns
  • Configure instance_maintenance_policy to control update behavior

Deployment

Quick Start

module "autoscaling" {
  source = "github.com/kogunlowo123/terraform-aws-autoscaling"

  name       = "my-asg"
  vpc_id     = "vpc-0123456789abcdef0"
  subnet_ids = ["subnet-abc123", "subnet-def456"]

  image_id      = "ami-0123456789abcdef0"
  instance_type = "t3.micro"

  min_size         = 1
  max_size         = 3
  desired_capacity = 2
}
